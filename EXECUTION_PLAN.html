§ 4.1Phase 1 · agi-sdk shell Band 1

P1 · W2-W3 Port the framework v0 SDK to agi-sdk
TLBE × 2QA
Deliverables
  • Lift sdk/bm_ai/ → packages/agi-sdk/agi/, rename namespace
  • sdk.models — role-keyed bindings, LiteLLM runner, ModelBinding.kwargs()
  • sdk.mcp — official MCP Python SDK wrapper
  • sdk.rag — Qdrant default, abstract VectorStore protocol
  • sdk.prompts — YAML loader, j2 rendering, locale resolution
  • sdk.config — Pydantic + layered loader (env > pack > operator > defaults)
  • sdk.serve — FastAPI + MCP exposure, baggage set on every request
  • sdk.trail — audit sink interface (memory / file-jsonl / mongo)
  • Traceloop.init() bootstrap → OpenLLMetry auto-instrument at module import
  • Pack loader: YAML → immutable Pack object
  • packs/blank/ reference pack
  • test_isolation_gate.py — fails CI if SDK imports agi_runtime/agi_ui
  • Lint rule: import openai/import anthropic/import boto3 requires # bm-ai: allow native sdk waiver comment
  • Publish to TestPyPI as agi-sdk==0.1.0
Acceptance criteria
  • pip install agi-sdk from TestPyPI on a clean VM succeeds in < 60s
  • 20-line main.py from examples/embedded runs against OpenAI/Ollama and emits OTel spans
  • Span attributes bm.pack, bm.use_case, bm.tenant_id overlay correctly
  • Switching operator.yaml's reasoning binding swaps provider with zero code change
  • Isolation gate test green; lint waiver enforced
  • ≥ 80% line coverage on each sdk.* module
  • Type-check passes under mypy strict mode

§ 4.1.5Phase 1.5 · Orchestrator adapters Band 1 · parallel with P1 tail

P1.5 · W3 Bless LangGraph + Pydantic-AI as optional
BE × 1
Deliverables
  • sdk.orchestrators.langgraph — ~150 LOC
  • sdk.orchestrators.pydantic_ai — ~150 LOC
  • Both expose resolve_tools(pack) and set_baggage(span_ctx)
  • Optional install extras: pip install agi-sdk[langgraph], pip install agi-sdk[pydantic-ai]
  • Reference use case using each adapter in examples/
  • Docs page: "Choosing an orchestrator" — plain async / langgraph / pydantic-ai decision tree
  • Tag v0.1.5 on TestPyPI
Acceptance criteria
  • Adapters add zero dependencies to base agi-sdk install
  • Trace shape (span attributes, ordering) is identical across plain-async, langgraph, pydantic-ai
  • Same example use case (e.g. bill_explainer) ships in all three flavours, identical behavior
  • Docs explicitly state when each is the right pick + watch-conditions for re-evaluation
Risks
LangGraph license drift

If langgraph-api tightens Elastic-2.0 boundary during this window, pause adapter and reassess. See PLAN.html § 5 watch-conditions.

§ 4.2Phase 2 · agi-mcpfyer Band 1

P2 · W4-W5 Generalise the auto-MCP-from-OpenAPI generator
TLBE × 1QA
Deliverables
  • Lift core/bm_core/mcpfyer/ + hub/ + registry/ → packages/agi-mcpfyer/
  • Generalise: parse any OpenAPI 3.0+ spec (not only TMF)
  • Per-tool metadata: input/output JSON Schema, side-effect flag, rate-limit class, dry-run support
  • One MCP endpoint per logical domain (configurable grouping)
  • Versioned tool bundles; SHA + semver pinning
  • CLI: agi-mcpfyer build path/to/openapi.yaml --out bundle/
  • Reference run: build MCP bundle from a public OpenAPI (e.g. Stripe, GitHub) and call a read tool end-to-end
  • Publish to TestPyPI as agi-mcpfyer==0.2.0
Acceptance criteria
  • Build a runnable MCP server from a non-TMF OpenAPI spec with zero code changes
  • Side-effect flag correctly inferred from HTTP method (overrides via OpenAPI extension)
  • Dry-run mode does not call upstream for write operations
  • Bundle reproducibility: same input → byte-identical output (SHA verified)
  • OpenLLMetry spans on every MCP tool call carry tool name + domain + side-effect flag
Risks
OpenAPI dialect variance

Real-world specs use OpenAPI 3.0 / 3.1 / Swagger 2 / various extensions. v1 supports 3.0+; document deviations in docs/mcpfyer-compat.md.

§ 4.3Phase 3 · agi-runtime Band 2

P3 · W6-W7 Reference FastAPI runtime + docker stack
TLBE × 2OpsQA
Deliverables
  • distribution/agi-runtime/ FastAPI app
  • Routes: POST /chat, GET/POST /tools, POST /tools/{name}, GET /kb, POST /kb, GET /trail, GET /trail/{cid}, POST /mcp, GET /healthz, GET /readyz, GET /admin/*
  • Claims-validated X-Pack header dispatch (mismatch → 401, missing → 400)
  • Pack-group-per-pod default; single-pack hardening mode behind env flag AGI_HARDEN=1
  • SSE streaming for /chat
  • distribution/agi-auth/ — Keycloak reference adapter, generic OIDC interface, static-token + dev-noop modes
  • Three-role RBAC enforced at dispatch (AGI_ADMIN, AGI_DEVELOPER, AGI_VIEWER)
  • deploy/docker/docker-compose.yml — agi-runtime + MongoDB + Qdrant + Langfuse + (optional) Ollama
  • Image published to GHCR: ghcr.io/<org>/agi-runtime:0.3.0
  • OpenAPI spec served at /openapi.json
Acceptance criteria
  • docker compose up on a clean VM brings the stack live in < 5 minutes
  • End-to-end /chat call with a real LLM succeeds and produces traces in Langfuse
  • X-Pack claim mismatch fuzz test: 1000 random claim/header pairs → 0 leaks past 401
  • Smoke: hot-swap a pack folder, POST /admin/packs/<slug>/reload, request succeeds with new config
  • OIDC adapter passes against a local Keycloak realm (sample realm shipped)
  • Hardening mode rejects all non-self-pack requests (single-pack-per-pod)
Phase gate 2

End of W7. TL + Ops sign off. If not green, P4 does not start.
