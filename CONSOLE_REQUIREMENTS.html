agi-ui · console requirements

Formal numbered requirements for the project-agi admin console. Each requirement has an ID, MoSCoW priority, traceability tags, and explicit acceptance criteria. Companion to ADMIN_CONSOLE.md (design narrative) and EXECUTION_PLAN.html § 4.4 (phase plan).

§ 1Scope

The admin console is part of the Reference Distribution (Band 2), not the SDK product. It ships in the agi-ui package and is optional — the runtime works headless. Day-to-day trace analysis lives in Langfuse; this UI handles what Langfuse cannot.

In-scope

  • Browsing and inspecting packs, tools, KB articles, use-case services, LLM role bindings, and AI-Trail events
  • Operator actions: pack reload, KB upload & reindex, tool test-invoke
  • Admin actions: user/scope mapping, settings, audit log read
  • Authentication via OIDC (Keycloak reference) or static-token (dev)

Out-of-scope

§ 2Conventions

RFC-2119 keywords

  • MUST the implementation has to satisfy this. Phase gate blocker.
  • SHOULD the implementation should satisfy this unless there is a documented exception.
  • MAY optional. Nice-to-have. Slip allowed.
  • WON'T out of scope for v1.0. May land later.

Requirement ID format

Each requirement is <CATEGORY>-<NN>. Categories follow ISO-style abbreviations:

  • FR-IA — Information Architecture
  • FR-AUTH — Identity, Authentication, Authorization (RBAC)
  • FR-PACK — Pack management
  • FR-TOOL — Tool catalogue
  • FR-KB — Knowledge Base
  • FR-TRAIL — AI-Trail (audit) viewer
  • FR-LLM — Model role bindings
  • FR-ADM — Admin pages (users, log, settings)
  • FR-INT — Backend integration contracts
  • NFR-* — Non-functional (PERF, SEC, A11Y, OBS, etc.)
  • EX-NN — Explicit exclusions

Traceability tags

Each requirement carries one or more tags pointing to source artefacts:

  • PLAN-§6 — section of PLAN.html
  • ADMIN-§3.4 — section of ADMIN_CONSOLE.md
  • DECISION-R3 — ratified decision in RESOLVED_STACK.md
  • DEBATE — surfaced in the framework/platform debate

§ 4Information Architecture · FR-IA

FR-IA-01
MUSTADMIN-§2
Provide a flat, role-aware top-level navigation

The console MUST render a single-level sidebar nav with items: Health, Packs, Tools, Use cases, Audit, LLM, Admin. Items the user lacks scope for MUST be hidden, not greyed.

Acceptance
  • Signed in as agi:operator:telco-demo: sidebar shows only Packs (auto-deeplinked) + Audit
  • Signed in as agi:admin: all items visible
  • No layout shift on signin/signout
FR-IA-02
MUSTADMIN-§3.1
Health dashboard at landing route

Route / MUST show service health rows for: agi-runtime, agi-core hub, vector store, Langfuse, each configured LLM provider, and storage. Each row MUST link to its detail page.

Acceptance
  • All rows render within 500ms of route mount
  • Health colours: green / amber / red; reading order matches PLAN.html § anatomy
  • Status text includes last-checked timestamp
FR-IA-03
SHOULDADMIN-§2
Auto-deeplink tenant operators

On signin, a user whose only scope is agi:operator:<slug> SHOULD be redirected to /packs/<slug>/overview instead of /.

Acceptance
  • Multi-pack operators land on a pack switcher chooser
  • Admins are not redirected (they keep /)
FR-IA-04
MUSTADMIN-§9
Empty and error states

Every page MUST render a meaningful empty-state and error-state. No raw 500/JSON dumps; no flicker between loading and error.

Acceptance
  • Zero packs deployed → packs route shows onboarding hint with docs link
  • Langfuse unreachable → health row turns amber with last-known timestamp
  • Insufficient scope → page renders "You don't have access to this page" with link back

§ 5Identity & RBAC · FR-AUTH

FR-AUTH-01
MUSTDECISION-R6PLAN-§9
OIDC sign-in via configured issuer

The console MUST authenticate users via the configured OIDC issuer (Keycloak as reference adapter). The console MUST NOT mint or hold password credentials.

Acceptance
  • OIDC callback exchanges code for tokens; tokens stored httpOnly
  • Refresh-token rotation handled silently while session is active
  • Local Keycloak sample realm shipped for dev
FR-AUTH-02
MUSTPLAN-§6DECISION-R1
Three-role RBAC enforced server-side

The runtime MUST enforce scopes: agi:admin, agi:operator:<slug>, agi:dev, agi:viewer. The console hides nav for scopes the user lacks; the runtime rejects requests for scopes the user lacks. Both required — UI hiding is never the security boundary.

Acceptance
  • Direct URL nav to a forbidden route returns the empty-state, not a 200 with data
  • Direct API call with insufficient scope returns 403, not 200
  • Penetration test: tampered scope claim is rejected at JWT verify
FR-AUTH-03
MUSTDECISION-R1DEBATE
Claims-based pack scoping

Operator scopes MUST encode the pack slug they apply to. The runtime MUST validate the requested X-Pack against the user's tenant claim on every request; mismatch returns 401.

Acceptance
  • Fuzz test: 1000 random claim/header pairs → 0 leaks past 401
  • Token tenant claim is sole source of truth; header alone never determines pack
FR-AUTH-04
SHOULDADMIN-§5
Static-token + dev-noop modes

The auth adapter SHOULD support a static-token mode (single token, fixed scope) and a dev-noop mode (anonymous, full scope) for local dev. Production deploys MUST NOT enable dev-noop; image MUST refuse to start with AGI_AUTH_MODE=dev-noop if AGI_ENV=production.

Acceptance
  • Image start with prod+dev-noop combo emits an error and exits non-zero
  • Docs explicitly warn about dev-noop

§ 6Pack management · FR-PACK

FR-PACK-01
MUSTADMIN-§3.2
Pack list and detail

Route /packs MUST list all packs the user has scope to see, with identity (slug, display name, vertical), source path, and SHA. Route /packs/:slug/overview MUST show identity, theme preview, role bindings, tool allow-list, and recent activity stats.

Acceptance
  • Pack list table virtualised; scrolls smoothly with 100+ packs
  • Theme preview shows the active pack's theme.primary as a swatch with hex
  • Recent activity stats refresh every 30s without full reload
FR-PACK-02
MUSTPLAN-§7DECISION-R3
Pack reload action

Pack overview MUST expose a "Reload pack" button that calls POST /admin/packs/:slug/reload. Action MUST be gated on agi:admin or agi:operator:<slug>. Outcome MUST be logged in admin log within 1 second.

Acceptance
  • Success toast within 2s of click
  • Failure shows the runtime error body (RFC-9457 problem-details)
  • Operator scoped to a different pack cannot trigger the action
FR-PACK-03
MUSTDECISION-R3DEBATE
Prompts shown read-only

Pack detail MAY show prompts; if shown, the UI MUST render them read-only with no edit, save, or delete affordance. The data source is the active pack's filesystem; runtime-edit endpoints MUST NOT exist.

Acceptance
  • Prompts page has no form controls, only a syntax-highlighted viewer
  • Backend has no POST /prompts or PATCH /prompts route
  • Audit reviewer signs off: no path to runtime prompt mutation
FR-PACK-04
SHOULDPLAN-§7
Hotfix lane visibility

Pack detail SHOULD show the latest hotfix branches (read from GET /admin/packs/:slug/hotfix-status) with merge status and time-since-deploy.

Acceptance
  • Shows up to 5 most recent pack-hotfix/* branches
  • Time-since-deploy refreshes without manual reload

§ 7Tool catalogue · FR-TOOL

FR-TOOL-01
MUSTADMIN-§3.3
Cross-pack and pack-scoped tool catalogues

Route /tools MUST list every tool emitted by agi-mcpfyer. Route /packs/:slug/tools MUST show only the tools allow-listed for that pack. Both views MUST display: name, domain, side-effect badge, rate-limit class, bundle version, and consuming-pack count.

Acceptance
  • Search + domain filter + side-effect filter all apply client-side
  • Side-effect badge: read / write distinguishable to colour-blind users
  • Bundle version is clickable to bundle changelog
FR-TOOL-02
MUSTADMIN-§3.3
Tool detail with JSON Schema

Route /tools/:name MUST render the tool's input and output JSON Schema, source OpenAPI op, side-effect flag, rate-limit class, dry-run support, and last build SHA.

Acceptance
  • JSON Schema collapsible by nested object
  • "Copy as MCP call" button copies a curl-shaped request to clipboard
FR-TOOL-03
MUSTADMIN-§3.3
Test-invoke with schema-driven form

Tool detail MUST expose a "Test invoke" panel that renders a form from the input JSON Schema (via Zod). Submitting calls POST /tools/:name and shows the response + a link to the resulting AI-Trail entry by correlation_id.

Acceptance
  • Required fields validated client-side before submit
  • Write/side-effecting tools require an explicit confirm step
  • Returned correlation_id is a clickable link to /audit/:cid
  • Failure shows RFC-9457 problem-details JSON with surface-level summary
FR-TOOL-04
SHOULDADMIN-§3.3
Dry-run mode for write tools

For tools with side_effecting: true and dry_run_supported: true, the test-invoke panel SHOULD offer a "Dry run" toggle that calls the tool with X-Dry-Run: 1. Default state is dry-run ON.

Acceptance
  • Toggle state persists per-session (not per-tool)
  • Visual indicator on the button differentiates dry-run from live

§ 8Knowledge Base · FR-KB

FR-KB-01
MUSTADMIN-§3.4
Per-pack KB browser

Route /packs/:slug/kb MUST list KB articles for the pack with: title, source format, size, chunk count, last-reindex timestamp.

Acceptance
  • Search bar performs vector + text hybrid search via runtime
  • Stale-index warning if last reindex > 24h
FR-KB-02
MUSTADMIN-§3.4
KB article upload

The KB browser MUST allow uploading markdown or JSON files via POST /kb (multipart). Upload MUST be gated on agi:operator:<slug> or agi:admin.

Acceptance
  • Drag-and-drop + click-to-pick both work
  • Files validated as text/markdown or application/json before send
  • Successful upload appears in list without manual refresh
FR-KB-03
MUSTADMIN-§3.4
Reindex action

KB browser MUST expose a "Reindex now" action that calls POST /admin/kb/:slug/reindex and shows progress. Action MUST be auditable.

Acceptance
  • Progress bar updates from a server-sent event stream
  • Result includes total chunks indexed and elapsed time
  • Failure does not block other actions on the page
FR-KB-04
SHOULDADMIN-§3.4
Article preview & per-article actions

KB browser SHOULD allow previewing an article's rendered content and supporting per-article re-embed and delete actions.

§ 9AI-Trail viewer · FR-TRAIL

FR-TRAIL-01
MUSTADMIN-§3.5DECISION-R5
Audit list with filters

Route /audit MUST list AI-Trail events filterable by pack, event type, and time range. The list MUST be virtualised; default time range is last 24h.

Acceptance
  • Scrolls smoothly at 100k events
  • Filter changes do not reset scroll position
  • URL reflects active filters (deep-linkable)
FR-TRAIL-02
MUSTADMIN-§3.5
Correlation_id detail page with event tree

Route /audit/:correlation_id MUST render the full event tree for one agent run, in chronological order, with per-event payload viewable.

Acceptance
  • Event tree renders even for runs with 500+ events
  • Payload viewer copy-to-clipboard works on every event
  • Side-effect events visually distinct (red dot)
  • Tool-call events link to the tool detail page
FR-TRAIL-03
SHOULDADMIN-§3.5
Deep-link to Langfuse for engineering traces

The correlation_id page SHOULD include a button "Open same run in Langfuse" that deep-links to the Langfuse trace search filtered by the same correlation_id.

Acceptance
  • If Langfuse is unreachable, the button is disabled with explanation tooltip
  • Langfuse endpoint URL comes from operator settings, not hard-coded
FR-TRAIL-04
MAYADMIN-§3.5
CSV export of filtered audit list

The audit list MAY support exporting the filtered event set as CSV for regulator reports.

§ 10Model role bindings · FR-LLM

FR-LLM-01
MUSTADMIN-§3.6DECISION-R4
Read-only role-to-model view

Route /llm MUST show all configured roles with their bound model_id, region, default params, and provider health. The view MUST NOT permit edits; this is a read surface.

Acceptance
  • Operator config is the source of truth; UI never writes to it
  • Provider health pings refresh every 30s; failures shown amber/red
FR-LLM-02
SHOULDADMIN-§3.6
Session-scoped LLM override for admins

Admins SHOULD be able to set a per-session X-LLM-Override header value via a "Rebind for this session" control. Override MUST NOT persist beyond the session and MUST apply only to that admin's subsequent calls.

Acceptance
  • Override is cleared on logout
  • UI banner indicates an override is active
  • Override events are in admin log

§ 11Admin · FR-ADM

FR-ADM-01
MUSTADMIN-§3.8
Admin action audit log

Every write action initiated from the console (pack reload, KB upload, KB reindex, LLM override) MUST appear in /admin/log within 1 second of completion. Log entries MUST include actor identity, action, target, timestamp, and result code.

Acceptance
  • Log is append-only at the runtime layer
  • Log entry latency < 1s in 99th percentile
  • Failed actions appear with their error
FR-ADM-02
MUSTADMIN-§3.8
Users / scope mapping view

Route /admin/users MUST list OIDC-issued identities currently active, with their resolved scopes. The view MUST NOT permit edits; user/scope provisioning lives in the OIDC issuer (Keycloak), not here.

Acceptance
  • Identity rows refresh every 60s
  • Scopes column displays scope strings as chips
FR-ADM-03
SHOULDADMIN-§3.8
Operator settings page

Route /admin/settings SHOULD show the operator-level configuration (Langfuse URL, vector-store URL, OIDC issuer, telemetry sampling) read-only. The view links to the YAML location for edits.

§ 12Non-functional · NFR-*

Performance · NFR-PERF

NFR-PERF-01
MUST
First paint < 1s on cable

Initial paint for / MUST be ≤ 1000ms on a 5 Mbps connection from a cold cache. Bundle size MUST be ≤ 250 KB gzipped on initial chunk.

NFR-PERF-02
MUST
Image size < 60 MB

The published agi-ui container image MUST be ≤ 60 MB compressed.

Accessibility · NFR-A11Y

NFR-A11Y-01
MUST
Lighthouse a11y score ≥ 95

Every top-level route MUST score ≥ 95 on Lighthouse accessibility. WCAG 2.1 Level AA contrast on all text. All interactive elements keyboard-navigable. Focus visible.

Security · NFR-SEC

NFR-SEC-01
MUST
No client-side secrets

The console MUST NOT embed API keys, signing secrets, or service credentials in the client bundle.

NFR-SEC-02
MUST
HttpOnly + Secure session cookies

Session cookies MUST be httpOnly, Secure, SameSite=Lax. Refresh tokens MUST NOT be exposed to JavaScript.

NFR-SEC-03
MUST
CSP enforced

The console MUST set a Content-Security-Policy header that disallows inline scripts (except hashed/nonced where unavoidable) and limits connect-src to the configured runtime origin.

NFR-SEC-04
SHOULD
Brute-force protection on the runtime

The runtime SHOULD rate-limit OIDC callback failures per source IP. The console SHOULD show a generic error for repeated failures, not signal which step failed.

Observability · NFR-OBS

NFR-OBS-01
MUST
Console emits structured logs

Server-side console code MUST emit structured JSON logs with request-id, route, scope, status, latency.

Theming · NFR-THM

NFR-THM-01
MUSTADMIN-§6
No hex literals in components

Components MUST consume CSS custom properties resolved from the active pack's theme block. Build MUST fail if a component contains a hex literal outside the token definitions file.

NFR-THM-02
SHOULD
Dark/light follows prefers-color-scheme

Theme SHOULD respect @media (prefers-color-scheme) by default with a user override stored per-session.

I18N · NFR-I18N

NFR-I18N-01
SHOULD
i18n hook in place; English-only at v1

The console SHOULD use next-intl (or equivalent) for all user-facing strings, even if only English is shipped at v1.

Browser support · NFR-CMP

NFR-CMP-01
MUST
Latest two versions of Chrome, Firefox, Safari, Edge

The console MUST function on the latest two versions of each major browser. Lower-than-required versions MAY show a soft warning.

§ 13Integration contracts · FR-INT

FR-INT-01
MUSTADMIN-§4
REST contract served by agi-runtime

All console API calls MUST go through the agi-runtime endpoints listed in ADMIN_CONSOLE.md §4. The console MUST NOT call vector stores, Mongo, or LLM providers directly.

FR-INT-02
MUST
Errors as RFC-9457 problem-details

Backend errors MUST be returned as RFC 9457 problem-details JSON. The console MUST surface title and detail verbatim; type is shown as a "More info" link.

FR-INT-03
MUST
OpenAPI spec drives client codegen

The runtime MUST publish its OpenAPI spec at /openapi.json. The console's API client MUST be generated from this spec — no hand-written fetch wrappers.

FR-INT-04
SHOULD
Bundle separately from runtime

The console SHOULD ship as a separate container image. Operators who don't deploy the UI MUST be able to skip it.

§ 14Excluded · EX-NN won't ship in v1.0

EX-01
WON'TDECISION-R3DEBATE
Runtime prompt editing

The console MUST NOT support editing prompts at runtime. This is a deliberate exclusion in service of the auditability principle (DECISION-R3). The hotfix lane (PLAN-§7) is the supported alternative.

EX-02
WON'T
Trace browser inside the console

Full trace span trees, token usage charts, latency dashboards — these live in Langfuse. The console deep-links instead of duplicating.

EX-03
WON'T
Eval runner UI

Eval is Promptfoo CLI in CI. v1.0 ships no UI for evals.

EX-04
WON'T
End-user chat surface

The end-user agent chat lives in the use-case service's own HTTP/MCP, not here.

EX-05
WON'T
Real-time event stream UI

Live event tailing is Langfuse's job. The console reads via REST polling.
